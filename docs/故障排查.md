# 故障排查

先保留控制台中第一条明确错误，确认任务 ID 和输入文件名，再按下面类别处理。不要把包含 token 的完整请求或日志直接发到公开 issue。

## API 返回 500、502、503 或 504

这通常是服务端、网关、模型名或协议路径问题，不是切片时间轴本身的问题。

1. 检查 `api_config.json` 的 `base_url` 是否为 API 根地址，不要重复写 `/chat/completions` 或 `/messages`。
2. 确认 `api_type` 与服务商协议一致：OpenAI 兼容用 `openai`，Anthropic 兼容用 `anthropic`。
3. 确认 token、模型名和账户额度有效；token 只在本机配置，不要复制到日志。
4. 检查 `AUTOSLICE_ANALYSIS_MODEL` 和 `AUTOSLICE_LLM_MODEL` 是否是服务商实际支持的模型。
5. 稍后重试。程序会对临时 502/503/504 做有限退避，但不会无限重试，也不会把 API 失败伪装成“0 个高光”。

如果预检连续失败，先在本机用服务商提供的最小请求测试 URL、协议和模型，再回到网页重跑。更换模型或提示策略后，旧的候选复核检查点会因策略版本变化而重新复核。

## FunASR 模型下载失败

首次转录需要下载模型。确认网络可访问 ModelScope，并在首次运行前允许下载：

```powershell
$env:MODELSCOPE_LOCAL_ONLY = "0"
$env:AUTOSLICE_FUNASR_DEVICE = "cpu"
python 启动.py
```

下载完成后可以改回 `MODELSCOPE_LOCAL_ONLY=1` 离线使用。若网络中断，清理不完整的模型缓存后重试；已有同名前缀 SRT 时，也可以先使用现有字幕跳过转录。

## GPU 不可用或 CUDA 报错

GPU 不是必需条件。先强制 CPU 验证业务链路：

```powershell
$env:AUTOSLICE_FUNASR_DEVICE = "cpu"
$env:AUTOSLICE_DISABLE_GPU = "1"
python 启动.py
```

若 CPU 正常，说明问题在 CUDA/PyTorch/驱动组合。检查 `nvidia-smi` 和当前 Python 的 `torch.cuda.is_available()`，不要为了切片临时修改系统驱动。隔离 GPU 运行时不存在或健康检查失败时，启动器会回到当前 Python；这不会影响 AutoCover。

## 端口 5002 或 5010 被占用

启动器会识别兼容的已有服务；不兼容服务可能导致端口顺延或启动失败。查看占用进程：

```powershell
Get-NetTCPConnection -LocalPort 5002,5010 -State Listen |
  Select-Object LocalPort,OwningProcess
Get-Process -Id <进程号>
```

确认是本项目残留进程后再停止：

```powershell
Stop-Process -Id <进程号>
```

不要盲目结束不认识的进程。再次执行 `python 启动.py`，看到 AutoSlice 和 AutoCover 的服务契约后再打开网页。

## 找不到 ffmpeg 或 ffprobe

运行：

```powershell
Get-Command ffmpeg
Get-Command ffprobe
```

如果找不到，把 FFmpeg 的 `bin` 目录加入用户或系统 `PATH`，重新打开 PowerShell。候选帧提取、精确切片、字幕预览和压制都会依赖这两个程序。

## 扫描不到视频

- 确认页面中的目录是实际成品目录，而不是父目录下的快捷方式；
- 默认话题分析扫描 `recordings/`，AutoCover 默认扫描 `output/`；
- 确认扩展名是 `.flv`、`.mp4`、`.mkv`、`.mov` 或 `.avi`；
- 检查是否把视频放进了被忽略的 `封面`、`视频素材` 子目录；
- SRT/ASS 和视频最好使用相同文件名前缀；
- 目录路径包含空格或中文时，直接在网页选择，不要手工拼接 JSON 字符串。

## 人工时间轴对不上

人工时间轴是北京时间，视频报告是视频内秒数。确认录播文件名带有正确的开播日期和时间，并确认时间轴没有把“现实钟点”误写成“播放进度”。多段录播时，分别选择对应视频和人工时间轴；优化功能会按字幕寻找语义邻近位置，但仍应抽查边界。

## 字幕 AI 校对失败或反复重试

AI 校对只处理结构化建议，不会覆盖原 SRT。检查字幕编码、序号、时间轴顺序和空字幕；源 SRT 在检查过程中被其他程序修改时，任务会主动失败并要求重新检查。确认 API 可用后，使用“重新检查”会按当前字幕指纹和校对策略生成新结果，旧缓存不会被错误复用。

## 字幕字体或压制失败

字幕默认要求精确字体 `Noto Sans S Chinese Black`。该字体不随仓库分发，需要在本机由 FFmpeg/libass 找到对应字体；程序会先验证解析到的字体家族。确认字体后再检查：

- 输出目录不是原视频目录或同名覆盖目标；
- `ffmpeg` 能读取输入视频和音频流；
- GPU 编码器失败时是否已自动回退 `libx264`；
- 磁盘空间是否足够写入临时 `.part.mp4`。

## AutoCover 贴图、字体或扫描失败

检查 `AUTOCOVER_INPUT_DIR`、`AUTOCOVER_OUTPUT_DIR`、`AUTOCOVER_STICKER_DIR` 和 `AUTOCOVER_FONT_PATH` 是否指向存在的本机目录/文件。贴图只扫描常见图片格式；字体缺失时可以先用系统回退验证界面，但正式导出前应配置濑户体或其他有授权字体。AutoCover 不需要 API 配置。

## 仍无法解决

收集以下脱敏信息：Windows/Python 版本、执行的入口、失败步骤、HTTP 状态码、`ffmpeg -version` 的版本号和不含 token 的错误摘要。不要上传录播、字幕、弹幕、DOCX、字体、贴图、cookies 或 `api_config.json`。
