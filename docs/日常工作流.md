# 日常工作流

本项目的目标是减少“从整场录播里找片段”和“反复手动核对边界”的时间。建议每次按下面顺序处理，保留原录播和原字幕，所有生成物写入单独目录。

## 1. 准备录播文件

把录播放入 `recordings/`，推荐让视频、SRT 和 ASS 使用相同的文件名前缀：

```text
recordings/
├── 示例直播-001.flv
├── 示例直播-001.srt
└── 示例直播-001.ass
```

SRT 没有时可以让 FunASR 自动生成；ASS 用于弹幕密度和峰值分析。分段录播可以分别扫描，文件名中的开播时间用于和人工时间轴对齐。

## 2. 选择人工时间轴（可选）

在 `topic-v2` 页面扫描后，可以明确选择本次使用的 DOCX，而不是让程序盲猜目录里的文件。人工时间轴的时间是北京时间，程序负责换算成视频内秒数；它是参考线索，不是剪切指令。

如果朋友的时间轴很详细，先使用“优化人工时间轴”功能：选择视频、SRT 和 DOCX，让 AI 把墙钟时间对齐到字幕语义，并拆分跨录播文件的记录。优化后的 JSON 仍然要经过字幕和弹幕核对，不能直接替代分析。

## 3. 运行话题分析

在 `http://127.0.0.1:5002/topic-v2`：

1. 扫描录播目录，确认视频、SRT、ASS 和人工时间轴路径。
2. 选择“无时间轴”或指定的人工时间轴文件。
3. 启动完整分析；没有 SRT 时先等待 FunASR。
4. 等待首轮整场分块分析、弹幕峰值整理和候选复核完成。
5. 打开生成的 `*_话题分析.md`，先看逐话题时间轴，再看带 `✂` 的候选。

报告中的时间是视频内播放时间。标题会尽量使用账号既有格式和音音、音姐等自然称呼；字幕无法确认的 SC 会通过前后语句、礼物/醒目留言关键词和弹幕证据辅助判断，但不会把模型猜测当成事实。

## 4. 理解自动切片结果

自动切片遵循以下优先级：

- 弹幕峰值只负责发现可能的高能位置；会读取峰值中的弹幕原文，区分具体互动、问号和无意义复读。
- AI 会检查峰值前的触发、音音的回应、观众 SC 的起点、事件收尾和话题自然边界。
- 片段不按每小时固定数量凑数；一个小时可以没有片，也可以有多个真正成立的片。
- 普通完整话题仍可写入报告，但不一定达到投稿价值门槛。
- 人工星标只提供有限加分；一般星标不能强行切片，星标很多时也必须有字幕和事件证据。
- 最终切片不重复、不重叠，通常保留必要的前因和结尾；过长话题会拆成相互独立的片段。

如果仍需人工精调，优先检查片段最开始是否包含 SC 或触发句，最后是否包含音音回应和观众反应。不要只依据 `start` 前后拖动几秒，要回到原录播确认完整语境。

## 5. 字幕校对与压制

把剪映导出的字幕和对应成品切片放入 `submissions/`，在 `subtitle-workflow` 页面扫描：

1. 选择视频和同名字幕，查看原始字幕内容。
2. 点击 AI 检查。AI 只返回原文、建议文字、依据和置信度，不直接覆盖 SRT。
3. 逐条人工确认，重点检查“音音/英英”、SC、专名、游戏名和观众昵称。
4. 保存校对字幕到新文件，再生成预览。
5. 确认预览无误后压制成新的 MP4，原视频不会被覆盖。

默认字幕样式是：

| 项目 | 默认值 |
|---|---|
| 字体 | `Noto Sans S Chinese Black` |
| 字号 | `20` |
| 字体颜色 | `ffffff` |
| 描边颜色 | `d06e95` |
| 描边粗细 | `100` |
| 位置 | `X=0, Y=-788` |
| 输出 | `1920x1080`、60 FPS、H.264 MP4、8000 kbps、Rec.709 SDR、音频复制 |

程序会先确认 FFmpeg/libass 能加载指定字体。字体无法精确解析时应先解决字体环境，不要默默接受错误字体。自动编码优先尝试 `h264_nvenc`，NVENC 不可用或失败时自动回退 `libx264`；回退后会重新检查时长、分辨率、帧率、音频和色彩信息。

## 6. AutoCover 制作封面

启动器启动两个服务后，打开 `http://127.0.0.1:5010`，或从 AutoSlice 页面进入 AutoCover。输入目录默认是 `output/`，也可以扫描精调后的成品目录。

建议使用最终成品视频抽帧。工具会避开片头、片尾和高风险字幕区域，再给出候选帧；仍应人工确认画面没有遮挡、表情不在过渡帧中。

在工作台中可以：

- 删除不相关的左侧切片任务；
- 拖动标题和贴图，鼠标缩放，不必手填坐标；
- 选择模板、常用文字颜色和独立描边颜色；
- 勾选双比例同步，让 `4:3` 与 `16:9` 一起应用修改，也可以取消后分别精调；
- 单独导出当前比例，或一次导出两个比例。

封面默认导出到 `covers/`，文件名会保留切片名称并追加 `-4x3.jpg` 或 `-16x9.jpg`。濑户体需要通过本机合法字体文件配置，贴图库需要通过 `AUTOCOVER_STICKER_DIR` 指向本机目录。

## 7. 投稿前检查

建议按下面清单快速复核：

- 视频开头是否有完整触发句，结尾是否保留回应或笑点；
- 片段之间是否有重复或重叠；
- 字幕中的音音、SC、专名和人名是否正确；
- 字幕位置是否遮挡表情或游戏关键画面；
- 封面比例、标题可读性、描边和贴图是否符合账号风格；
- B 站投稿标题、简介、标签和封面是否对应同一个话题。

AutoSlice 会生成精调任务清单，便于把“核查前后文、剪气口、字幕、片头片尾、导出、封面、投稿”逐项完成。重跑同一任务时，程序会清理旧的自动生成片段，但尽量保留用户手工命名的文件；重要精调成品仍建议放在独立目录。
