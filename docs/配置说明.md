# 配置说明

本文是参考手册，说明公开版的目录、模型、时间轴、FunASR、字幕和 AutoCover 配置。所有示例都使用占位值，请在本机替换，不要把真实 token 写进 Git。

## 1. 安装依赖

在项目根目录执行：

```powershell
python --version
python -m pip install -r requirements.txt
```

`requirements.txt` 包含 Flask、FunASR、python-docx、Pillow 和 requests。`ffmpeg` 与 `ffprobe` 不是 Python 包，必须单独安装并加入 `PATH`：

```powershell
Get-Command ffmpeg
Get-Command ffprobe
```

两个命令都能找到可执行文件后，视频扫描、候选帧提取和字幕压制才具备完整条件。

## 2. API 配置

复制示例：

```powershell
Copy-Item api_config.example.json api_config.json
```

最小配置如下：

```json
{
  "base_url": "https://api.example.com/v1",
  "token": "YOUR_API_TOKEN",
  "model": "YOUR_REVIEW_MODEL",
  "api_type": "openai"
}
```

### `base_url` 的填写规则

`base_url` 必须是 API 的 HTTP(S) 根地址，例如 `https://api.example.com/v1`。不要填写完整的 `/chat/completions`、`/messages` 或带 token 的 URL。程序会根据协议自动追加路径：

- `api_type: "openai"`：追加 `/chat/completions`，使用 `Authorization: Bearer ...`。
- `api_type: "anthropic"`：追加 `/messages`，使用 `x-api-key` 和 Anthropic 版本请求头。

服务商必须提供对应的兼容协议；“看起来像 OpenAI”的地址不代表一定支持 JSON 输出或指定模型。

### 模型分工

- `model` 是配置文件中的默认模型，主要用于候选复核、标题和最终投稿价值判断。
- `AUTOSLICE_ANALYSIS_MODEL` 控制首轮整场分块分析，默认是 `GPT-5.6-luna`。
- `AUTOSLICE_LLM_MODEL` 控制复核和需要更强判断的阶段，默认是 `GPT-5.6-Terra`，也会覆盖配置文件中的 `model`。

例如：

```powershell
$env:AUTOSLICE_API_BASE_URL = "https://api.example.com/v1"
$env:AUTOSLICE_API_TOKEN = "YOUR_API_TOKEN"
$env:AUTOSLICE_API_TYPE = "openai"
$env:AUTOSLICE_ANALYSIS_MODEL = "GPT-5.6-luna"
$env:AUTOSLICE_LLM_MODEL = "GPT-5.6-Terra"
python 启动.py
```

环境变量优先于 `api_config.json`。如果只设置了其中一个 API 环境变量，程序会要求另外的必需字段也完整提供。程序不会自动加载仓库中的 PowerShell 示例；请复制为已忽略的本机文件后再编辑和加载：

```powershell
Copy-Item environment.example.ps1 environment.local.ps1
notepad environment.local.ps1
. .\environment.local.ps1
```

token 只放在本地配置或当前 PowerShell 会话中；不要直接修改并提交示例文件，也不要把 token 写入 README、截图、日志或提交记录。

AutoSlice 不会读取 `Claude`、`Codex`、编辑器插件或其他项目的配置文件，也不会自动寻找别的 API key。发送给模型的内容只包括当前任务需要的字幕、有限弹幕摘要和可选时间轴线索。

## 3. 目录环境变量

默认目录都在项目根目录下：

| 环境变量 | 默认目录 | 用途 |
|---|---|---|
| `AUTOSLICE_VIDEO_DIR` | `recordings` | 录播扫描目录 |
| `AUTOSLICE_OUTPUT_DIR` | `output` | 报告、JSON、自动切片 |
| `AUTOSLICE_TIMELINE_DIR` | `timelines` | 人工时间轴 DOCX |
| `AUTOSLICE_SUBMISSION_DIR` | `submissions` | 字幕校对与投稿素材 |
| `AUTOSLICE_REFINEMENT_QUEUE_DIR` | `output` | 精调任务清单 |
| `AUTOSLICE_TITLE_STYLE_PROFILE` | 示例配置 | 私人标题风格样本 |
| `AUTOSLICE_STREAMER_PROFILES` | `streamer_profiles.json` | 主播称呼、标题前缀和 ASR 规则 |
| `AUTOCOVER_INPUT_DIR` | `output` | AutoCover 默认切片目录 |
| `AUTOCOVER_OUTPUT_DIR` | `covers` | AutoCover 封面输出目录 |
| `AUTOCOVER_STICKER_DIR` | `stickers` | 本机贴图库 |
| `AUTOCOVER_FONT_PATH` | 未配置 | AutoCover 本机字体文件 |

PowerShell 示例：

```powershell
$env:AUTOSLICE_VIDEO_DIR = "$HOME\直播录播"
$env:AUTOSLICE_OUTPUT_DIR = "$HOME\自动切片结果"
$env:AUTOCOVER_OUTPUT_DIR = "$HOME\封面"
$env:AUTOCOVER_STICKER_DIR = "$HOME\表情包"
$env:AUTOCOVER_FONT_PATH = "$HOME\字体\seto-bilibili.ttf"
python 启动.py
```

路径可以是绝对路径，也可以在代码中通过环境变量传入；公开仓库本身不预设任何个人盘符。标题风格需要私人样本时，复制 `title_style_profile.example.json` 为本地的 `title_style_profile.json` 后再修改，私有文件不会被 Git 跟踪。

## 4. 服务访问边界

默认启动不需要任何网络安全配置，AutoSlice 和 AutoCover 都只供本机访问。AutoSlice 默认绑定 `127.0.0.1`，并拒绝不受信任的 Host 与跨站写请求。

只有明确需要从局域网其他设备访问 AutoSlice 时，才启用高级 LAN 模式：

```powershell
$env:AUTOSLICE_LAN_MODE = "1"
$env:AUTOSLICE_LAN_TOKEN = "REPLACE_WITH_AT_LEAST_24_RANDOM_CHARACTERS"
$env:AUTOSLICE_LAN_HOSTS = "192.168.1.20"
$env:AUTOSLICE_LAN_ORIGINS = "http://192.168.1.20:5002"
$env:AUTOSLICE_ALLOWED_ROOTS = "$HOME\直播录播;$HOME\自动切片结果"
python 启动.py
```

- `AUTOSLICE_LAN_TOKEN` 至少 24 个字符，用于保护所有 LAN 请求；
- `AUTOSLICE_LAN_HOSTS` 是允许出现在请求 Host 中的主机名或 IP，可用分号分隔；
- `AUTOSLICE_LAN_ORIGINS` 是允许发起写请求的完整来源，包含协议和端口；
- `AUTOSLICE_ALLOWED_ROOTS` 限制 LAN 请求可扫描、读取或写入的目录；
- AutoCover 仍保持本机 loopback 服务，不应直接暴露到局域网。

LAN 模式不等于公网部署方案。不要直接做路由器端口映射；需要跨网络访问时，应使用带 HTTPS 和额外认证的反向代理或 VPN。

## 5. 主播识别与投稿标题前缀

自动模式会优先识别录播文件名中的主播名。推荐以下命名：

```text
泽音-2026-07-22 19_58-周三歌杂.flv
七海Nana7mi-2026年07月22日20点00分.flv
主播名_20260722_2000.flv
```

程序只在主播名后存在明确日期标记时提取文件名前缀，避免把普通视频标题随意当成主播名。已知主播会匹配 `streamer_profiles.json` 中的专属配置；未知主播会在当前任务中自动生成 `【文件名前缀】`，不会写回配置文件。模型返回了错误的旧账号前缀时，程序也会先删除旧前缀，再统一添加当前录播对应的前缀。

旧式录播文件名可能以直播标题开头。这种情况下可以在 `streamer_profiles.json` 的 `path_keywords` 中配置稳定的父目录关键词，或显式增加主播配置。自定义配置建议复制到本机其他路径，再设置：

```powershell
$env:AUTOSLICE_STREAMER_PROFILES = "$HOME\AutoSlice\streamer_profiles.local.json"
```

配置中的 `title_style_profile` 必须是相对于主播配置文件所在目录的路径。API token 不属于主播配置，不要写入该文件。

## 6. FunASR CPU/GPU

没有同名 SRT 时，AutoSlice 会提取音频并调用 FunASR。可显式选择设备：

```powershell
$env:AUTOSLICE_FUNASR_DEVICE = "cpu"
python 启动.py
```

可用 CUDA、PyTorch 和对应驱动时可以使用：

```powershell
$env:AUTOSLICE_FUNASR_DEVICE = "cuda:0"
python 启动.py
```

`auto` 会尝试使用启动器管理的隔离 GPU 运行时，无法确认健康时回退到当前 Python 的 CPU/默认路径。首次下载模型时允许 ModelScope 网络访问；模型已经缓存后可以设置 `MODELSCOPE_LOCAL_ONLY=1` 离线运行。GPU 不是硬性要求，CPU 会更慢但不改变分析逻辑。

## 7. 人工时间轴

人工时间轴记录的是北京时间（现实钟点），不是视频播放进度。例如录播文件名包含开播时刻时，程序会将时间轴的钟点减去开播时刻，换算成当前视频内的秒数。多段录播属于同一场直播时，程序会按每段文件的起止范围匹配记录；不在当前段的记录不会直接套用。

时间轴只提供主题、星标和人工线索。自动分析仍然以字幕和弹幕为主，并会重新核对实际发生的前因后果、SC 起始和自然边界。详细时间轴可以先通过“优化人工时间轴”功能结合 SRT 对齐，再作为辅助输入。

## 8. AutoCover 字体与贴图

AutoCover 默认尝试使用濑户体。该字体及 B 站网页字体不随本仓库再分发；请使用自己有权使用的字体文件，并通过 `AUTOCOVER_FONT_PATH` 配置。缺少时会显示系统回退状态，不能把回退字体误认为濑户体。

贴图放在 `AUTOCOVER_STICKER_DIR` 指向的目录下，工具会扫描常见 PNG/JPG 图片。不要把个人贴图或字体加入公开仓库。AutoCover 支持 `4:3`（1440x1080）与 `16:9`（1920x1080），可分别或同步调整标题、颜色、描边、位置和缩放。

## 9. 本地配置检查

启动前可以先检查：

```powershell
Test-Path api_config.json
Get-Command ffmpeg
Get-Command ffprobe
python -c "import flask, funasr, PIL, docx, requests; print('依赖正常')"
```

API 配置错误不会通过 AutoCover 解决；先看 AutoSlice 控制台中的预检错误，再参照[故障排查](故障排查.md)。
